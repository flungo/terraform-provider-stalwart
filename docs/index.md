# Stalwart Provider

The Stalwart provider manages configuration of a [Stalwart](https://stalw.art) mail and
collaboration server (v0.16+). It targets the server's JMAP management API to create,
read, update, and delete domains, accounts, groups, mailing lists, DKIM signatures, and
other objects.

## Example Usage

```terraform
terraform {
  required_providers {
    stalwart = {
      source = "flungo/stalwart"
    }
  }
}

provider "stalwart" {
  endpoint = "https://mail.example.com" # base URL; the provider appends /jmap
  token    = var.stalwart_token         # or set STALWART_TOKEN

  # Username/password authentication is an alternative to a bearer token:
  # username = "admin"
  # password = var.stalwart_password
}

variable "stalwart_token" {
  type      = string
  sensitive = true
}
```

## Authentication

The provider authenticates to the Stalwart JMAP endpoint using either a bearer token or
username/password credentials. Both may be supplied directly in the provider block or
via environment variables.

| Environment variable | Description                              |
|----------------------|------------------------------------------|
| `STALWART_ENDPOINT`  | Base URL of the Stalwart server          |
| `STALWART_TOKEN`     | Bearer token (preferred)                 |
| `STALWART_USERNAME`  | Username for basic authentication        |
| `STALWART_PASSWORD`  | Password for basic authentication        |

When both token and username/password are set, the token takes precedence.

### Optional

- `username` (String) Username for HTTP Basic authentication (alternative to `token`). May also be set with the `STALWART_USERNAME` environment variable.
